PlanSIO

Application de planning et de suivi des activités professionnelles pour les élèves de BTS SIO. Développée en équipe dans le cadre d'un projet transversal, elle centralise les rapports de stage, les compétences validées et les ressources pédagogiques dans une interface unifiée.

Contexte du projet

Pourquoi ce projet ?

PlanSIO répond à un besoin concret identifié au sein du BTS SIO : les élèves gèrent leurs activités professionnelles sur des supports dispersés (fichiers Word, Google Drive personnel, mails…), et les professeurs n'ont pas de vue centralisée sur la progression de leur classe. L'objectif était de créer un outil unifié couvrant la totalité du cursus.

Le projet a été développé en équipe de 3 personnes avec une organisation inspirée de Scrum : sprints de deux semaines, réunions de planification et rétrospectives. Chaque membre avait un domaine de responsabilité principal (backend, frontend, infrastructure) tout en contribuant à l'ensemble.

L'architecture choisie est une séparation stricte backend/frontend : un backend Symfony expose une API REST consommée par un frontend Vue.js. Cette organisation a permis de travailler en parallèle et de préparer une éventuelle ouverture vers une application mobile.

Fonctionnalités

Ce que fait l'application

Calendrier des activités
Vue mensuelle / hebdomadaire des activités professionnelles avec catégorisation par type (stage, projet, atelier).
Gestion de documents
Upload, versionnage et partage des rapports de stage avec historique des révisions et commentaires professeur.
Suivi des compétences
Tableau de progression par compétence du référentiel BTS SIO, avec auto-évaluation et validation professeur.
Messagerie interne
Système de messagerie élève ↔ professeur avec fils de discussion par document et notifications en temps réel.
Génération de rapports PDF
Export automatisé du bilan de compétences et du portfolio d'activités au format PDF prêt à imprimer.
Gestion multi-rôles
Trois niveaux d'accès : Élève, Professeur référent et Administrateur — avec interfaces distinctes.
Stack technique

Technologies utilisées

Le backend Symfony 6 gère la logique métier, l'API REST (API Platform) et l'authentification JWT. Le frontend Vue.js 3 (Composition API) consomme l'API avec Axios et gère l'état global avec Pinia. L'environnement de développement est entièrement dockerisé pour uniformiser les postes de l'équipe.

Défis & apprentissages

Ce que j'ai appris

Travail en équipe avec Git
Coordonner le travail de trois développeurs sur le même dépôt a nécessité une discipline rigoureuse : branches feature, pull requests obligatoires, revues de code. Les conflits de merge m'ont appris à mieux comprendre l'historique Git et à utiliser le rebase de façon sûre.
Architecture API REST avec Symfony
API Platform accélère le développement d'API mais impose ses conventions. Comprendre les groupes de sérialisation, les filtres et les voters de sécurité a demandé un investissement important mais m'a donné une base solide pour de futurs projets Symfony.
Versionnage de documents
Implémenter un système de versionnage simple sans Git côté applicatif a été un défi de conception : chaque upload crée une nouvelle version liée à la précédente. La gestion du stockage et du nommage des fichiers sur le serveur a demandé une réflexion soignée.
