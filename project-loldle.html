Contexte du projet

Pourquoi ce projet ?

LOLDLE est né de la volonté de combiner deux passions — le développement web et League of Legends — dans le cadre du projet E5 du BTS SIO SLAM. Le défi principal imposé par le cahier des charges était de rendre l'application utilisable sans connexion internet, ce qui a orienté l'ensemble des choix techniques vers l'architecture PWA.

L'application permet aux joueurs de tester leurs connaissances sur les champions, leurs sorts, leurs lores et leurs statistiques. Plusieurs modes de jeu sont disponibles : mode classique (QCM), mode silhouette (deviner le champion à partir de son ombre), et mode citation (reconnaître le champion depuis une phrase iconique).

Le backend Node.js expose une API REST qui alimente la base de champions, synchronisée avec les données officielles de Riot Games via leur API publique. Le frontend React consomme cette API et stocke les données localement dans IndexedDB pour garantir le mode hors-ligne.

Ce projet m'a permis de découvrir en profondeur le fonctionnement des Progressive Web Apps : cycle de vie des Service Workers, stratégies de cache (Cache First, Network First), synchronisation différée et installation sur écran d'accueil.
Fonctionnalités

Ce que fait l'application

Mode hors-ligne complet
Service Worker avec stratégie Cache First — toute l'app fonctionne sans connexion, y compris les questions et les images.
Synchronisation automatique
Les scores réalisés hors-ligne sont mis en file d'attente (Background Sync API) et envoyés au serveur à la reconnexion.
3 modes de jeu
Quiz classique, mode silhouette et mode citation — chaque mode cible des connaissances différentes sur les champions.
Base de données de champions
Plus de 160 champions avec leurs sorts, statistiques et lores, synchronisés depuis l'API Riot Games.
Classements & scores
Système de scoring avec combo multiplicateur, historique personnel et classement global en temps réel.
Partage de score QR Code
Génération d'un QR code unique à la fin de chaque partie pour partager son score avec d'autres joueurs.
Installable sur mobile
Manifest PWA configuré pour permettre l'installation sur l'écran d'accueil iOS et Android comme une app native.
Interface responsive dark
Design entièrement responsive inspiré de l'univers visuel de League of Legends avec thème sombre et effets lumineux.
Stack technique

Technologies utilisées

Le frontend React gère toute la logique de jeu côté client. Le routing est assuré par React Router et le state global par Context API. Le backend Node.js + Express fournit l'API REST pour les champions, les scores et l'authentification. La base de données MySQL stocke les questions, les utilisateurs et les classements.

Défis & apprentissages

Ce que j'ai appris

Stratégie de cache Service Worker
Choisir la bonne stratégie de cache selon la ressource (Cache First pour les assets statiques, Network First pour les données de score) a demandé beaucoup de tests. Une mauvaise stratégie rend soit l'app inutilisable offline, soit les données périmées.
Synchronisation différée des scores offline
Gérer la synchronisation des scores réalisés hors-ligne sans conflits avec les scores en ligne était complexe. J'ai implémenté un système de queue locale avec timestamp pour résoudre les conflits lors de la synchronisation.
Consommation de l'API Riot Games
L'API Riot impose des limites de taux (rate limiting) strictes. J'ai mis en place un système de mise en cache côté serveur pour éviter de dépasser les quotas tout en gardant les données champions à jour.
